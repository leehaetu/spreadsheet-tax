Software integrity statement

This product is built for HMRC Making Tax Digital bridging. Inspectors and customers should be able to tell exactly which parts talk to HMRC and which do not. Machine-readable copy: /api/integrity.

HMRC recognition

Not HMRC-recognised software. Spreadsheet Tax is not on HMRC’s recognised MTD software list. We develop against the sandbox and pilot controls only. We will not claim “HMRC recognised” or public live filing readiness until recognition is actually obtained. App version numbers (e.g. on /health) are our software release (semantic version), not an HMRC software ID or submission reference.

What is real today

  • Spreadsheet import — CSV/Excel parse, field mapping, digital-link traces, period payloads
  • Validation — blocks invalid periods, NINO, tax year, business IDs before submit
  • Server-owned drafts — import stores figures on the server; submit uses draftId
  • Preview submit (default) — builds the same HTTP request shape HMRC expects, stores a local receipt, does not call HMRC
  • Authenticated practice workspace — real SQLite firm/clients under /workspace
  • Auth sessions — register, login, password change/reset (email delivery is stubbed to logs)

What is not real HMRC filing

  • Preview / double mode — default public submit; labels say “not sent to HMRC”
  • Mock OAuth — without Developer Hub credentials, “connect” stores a local mock token only
  • Demo portfolio — /accountant and /practice use fictional in-memory data
  • Billing — plan selection is stored; cards are not charged
  • Email — invites/reminders/password reset are logged, not delivered by a mail provider

What HMRC live requires (not claimed complete)

  1. HMRC Developer Hub application registration
  2. HMRC_CLIENT_ID / HMRC_CLIENT_SECRET and real OAuth (HMRC_OAUTH_MOCK off)
  3. HMRC_ALLOW_LIVE_SUBMIT=1 on the server
  4. Non-mock access token on the user account
  5. Fraud-prevention header pack validated against current HMRC guidance
  6. Production approval for live filing

Until those are true, any successful “submit” in the public app is a preview.
