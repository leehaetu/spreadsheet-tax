Product path

How it works

Concrete steps — not abstract “digital transformation”.

1

Upload

CSV or Excel period summary (template available). Signed-in accounts store a server draft.

2

Check

See mapped totals for this period and year-to-date, with source cell references where available.

3

Approve

Explicit approval of the figure set. Re-upload invalidates prior approval.

4

Send

Preview by default (not HMRC). Sandbox/live only with OAuth + operator flags. Receipt stored on attempt.

Errors and modes

  • Validation failures show before submit (identifiers, mapping issues).
  • HMRC non-2xx responses are failures — not scored as success.
  • Preview / sandbox / live are labelled in the product status pill.
